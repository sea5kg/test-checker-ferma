
Универсальный сервис.

## Этап самоконфигурации сети.

1. Ноды сканять текущую сеть по текущему интерфейсу.
2. Если находит другую ноду подключается и рассылает уже существующим что усе есть.

Между каждой нодой есть два сокета - входящий и исходящий - по ним происходит обмен и синхронизация данными.

## Статусы

На каждой ноде есть интерфейс со списком нод и выполняемых задач (каждая нода сама информирует что происходит на ней).
(через веб интерфес)

## Постановка задачи

1. Список образов
  - загрузка нового образа и его импорт

2. Выполнение команд - список для запуска (синкается между всеми)

Количество задач может быть только по количеству ядер * 1.5 (настраиваемая штука)

## новая нода

Если вдруг появляется новая нода то она запрашивает текущие настройки (синхронизируется с остальными)
